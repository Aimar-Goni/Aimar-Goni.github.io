Vorpal Engine

C++ OpenGL VR

A proprietary engine written in C++ that uses OpenGL, and integrates VR, portals, and advanced rendering techniques. The engine was developed by Carlos Cordero and myself during our last year of an HND in Computer Science, for the subject of Engine Programming (8 months | October 2023 - May 2024). With its focus on graphics programming techniques, while being extremely user-friendly.

The main inspiration for the Vorpal Engine was the following video by CodeParade. Inspired by that demo, I wanted to create a game engine that could support portals as well as implement those portals in VR.

The finished project has both features working (although under some limitations). Here is how they work:

Portal Rendering works by rendering the scene from the point of view of the player as if placed on the other side of the portal. This process happens in each frame to render a scene with portals.

Portal Rendering Process:

  • The recursion level is set to the maximum of bounces allowed between portals.
  • An occlusion culling query is issued so the portals outside the main view don’t perform unnecessary draw calls.
  • A framebuffer renders the scene from the camera's perspective if it were on the other side of the portal, cropped to the size of the portal.
  • The process is repeated for any visible portals in the current framebuffer's scene.
  • The recursion continues until no portals are left or the recursion level reaches 0 (portals are shown with a pink texture if not rendered).

The VR rendering pipeline renders the scene twice from different offset points of view, using distinct MVP matrices for each eye.

VR Rendering Process:

Main VR View:

  • Each eye has its own framebuffer.
  • The scene is drawn twice with separate MVP matrices for each eye's perspective.
  • Controller models and rays are rendered on top of the textures, using OpenVR SDK.
  • Textures are submitted to the VR headset via the VR compositor.

Companion Window View:

  • Framebuffers are reused for both eye views in a split-screen display.
  • ImguiUI is rendered on top of the split-screen display.
